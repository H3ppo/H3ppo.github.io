比特币概念

比特币是一种基于密码学的电子货币，在2008年由中本聪在白皮书：“比特币：一种点对点的电子现金系统（Bitcoin: A Peer-to-Peer Electronic Cash System）”中提出（一种去中心化的电子记账思想）

区块链技术

比特币交易被记录在一个公开的分布式账本——区块链上。一旦交易被确认并加入到区块链中，它将被串联到前一个区块后面。每个区块包含了前一个区块的加密散列值（hash），这种链式结构确保了一旦信息被记录，就无法在不被发现的情况下被修改。

比特币网络功能上的特点：
  • 去中心化
  • 隐私性
  • 通胀预防
    比特币一条交易可能包括：
  • 付款人地址：合法的地址，公钥经过 SHA256 和 RIPEMD160 两次 Hash，得到 160 位 Hash 串；
  • 付款人对交易的签字确认：确保交易内容不被篡改；
  • 付款人资金的来源交易 ID：从哪个交易的输出作为本次交易的输入；
  • 交易的金额：多少钱，跟输入的差额为交易的服务费；
  • 收款人地址：合法的地址；
  • 时间戳：交易何时能生效。

    记账奖励 –为什么参与？
    ①手续费 ②打包奖励
    奖励方案是，每十分钟生成一个区块，每生成一个区块会奖励一定数量的比特币，最开始是50个BTC，过4年会奖励25个BTC，再过4年再减少一半，以此类推。这样比特币的产生会越来越少，越来越趋近于一个最大值，计算公式是：50×6×24×365×4×（1+1/2+1/4+1/8+…）≈2100万
    工作量证明（Proof of Work，PoW）–以谁为准？
    比特币使用工作量证明机制来添加新区块。矿工必须解决一个计算密集型的问题，这需要大量的计算资源。第一个解决问题的矿工可以将新的区块添加到区块链中。这个过程称为“挖矿”。工作量证明确保篡改历史区块是不切实际的，因为它需要重新计算所有后续区块的工作量证明，这在计算上是不可行的。
    挖矿原理
    挖矿是指网络中的维护节点，通过协助生成和确认新区块来获取一定量新增比特币的过程。
  5. 哈希函数
    sha256生成256位的二进制数，逆向困难
  6. 原理
    字符串：前块头部+账单+时间+随机数(猜测)
    hash = sha256(sha256(字符))
    要求前n位为0
    这代表各个挖矿者挖矿成功的难度不一（账单·时间·随机数都不一样）
  7. 挖矿成功的难度n确定：
    比特币设计者希望，总体上平均每十分钟产生一个区块，总体上来看，挖矿成功的概率为1/2^n。现假设世界上有1W台矿机，每台矿机的算力是14T次/s = 1.4×10^13次/s，单位次/s称之为哈希率，10分钟是600s，所以10分钟可以做8×10^19次哈希运算，从概率角度看，想要挖矿成功需要做2^n次运算，可以列出等式2^n = 8×10^19，可以解出n约为66。因此要提高算力，才能尽快算出结果

    区块链技术分类

    根据参与者，区块链可以分为
  • 公有（Public 或 Permissionless）链
  • 联盟（Consortium 或 Permissioned）链
  • 私有（Private）链
    一般来说，去中心化程度越高，信任度越好。但两者的关系并非线性那么简单。随着节点数增加，前期的信任度往往会增长较快，到了一定程度后，信任度随节点数增多并不会得到明显改善。这是因为随着成员数的增加，要实现共谋作恶的成本会指数上升。

    比特币安全问题

    一、身份认证 –电子签名技术

    比特币采用电子签名技术防止伪造。
    生成公钥私钥

1.用公钥加密的数据，只能被之恶个公钥对应的私钥解密
2.用私钥签名的数据，可以被这个私钥对应的公钥验证

    A[随机数]
    B[私钥]
    C[公钥]
    A --> B
    B --> C
    C --> D[地址]

对原始消息进行加密

graph LR
    A[原始消息]
    B[摘要]
    C[密码]
    A -->|hash| B
    B -->|私钥| C

广播密文与原始消息

graph LR
    A[密码]
    C[原始消息]
    A -->|公钥解密| F[摘要1]
    C -->|hash| G[摘要2]
    F --> H{摘要1==摘要2?}
    G --> H
    H -->|一致| I[true]
    H -->|不一致| J[false]

若摘要一与摘要二相等，则认为消息为真
在此处私钥可以防止消息伪造，只在广播者手中。

二、双重支付问题

余额检查

追溯
若A获得了50个比特币，转账给B 30个比特币。
A再转给B 10个比特币可认为真，A转给B 60比特币可认为假。

双重支付

A只拥有10个比特币，同时转账给B和C二人10个比特币并广播，则后面那条链成功上链（挖矿成功），视为该链有效（最长链原则）。

graph LR
    A[A 10BTC] --> B[A->B 10BTC]
    A --> C[A->C 10BTC]
    B --> D[D,E,F（F挖矿成功，视为此链有效）]
    C --> E[G,H（由于F先挖矿成功，会跟在F后挖矿，视为此次交易无效）]

三、防止篡改

最长链原则

比特币的最长链原则是指在多个区块链分支出现时，具有最多区块的链被认为是有效的主链。这个原则是比特币网络达成共识的机制之一。

当比特币网络中的不同矿工几乎同时发现新区块时，会暂时出现区块链的分叉。网络上的节点（包括矿工和其他参与者）必须决定哪个区块链分支是有效的。按照最长链原则，节点应选择最“长”的链，即包含最多区块的链，作为主链并继续在其上构建新的区块。

这个原则背后的理念是，最长的链表明其上的工作量（或者说计算量）最大，因为比特币网络通过工作量证明（Proof of Work）算法来创建区块，所以最长链代表了最大的工作量共识。它也最有可能代表了网络大多数算力的意志，因此更加安全和难以篡改。

graph LR
    A[区块1] --> B[区块2]
    B --> C[区块3A]
    B -->|X| D[区块3B]
    C --> E[区块4]

除非能使3B链比3A链更长，否则3B链无效。

防止篡改

若一个人（组织）有算力能造出比上面更长的链，那么他在长链接着算会有更多收益（博弈论：成本巨大，代价超过好处）。

比特币的创新点

比特币的创新点主要是避免作恶、采用负反馈调节和基于概率的共识机制等三个方面。

避免作恶

已经在上文安全问题的防止篡改中提及

负反馈调节

  • 矿工越多，系统越稳定，比特币价值越高，挖到矿改率降低；
  • 矿工越少，系统易受攻击，比特币价值越低，挖到矿概率提高。
    预计比特币将在2140年达到2100万枚，挖矿奖励（打包费）耗尽，只能依靠交易的手续费来鼓励矿工维护比特币系统。

    共识机制

    工作量证明
    由于比特币网络完全开放，易受攻击，受网络质量影响，所以提出了工作量证明（Proof of Work，PoW）的共识机制。
  1. 不实现面向最终确认的共识，而是基于概率、随时间逐步增强确认的共识。现有结果在理论上可能被推翻，但是代价巨大，随时间上升，可能性下降。
  2. 网络大，达成共识时间长。按照区块进行阶段性确认，提高网络整体可用性。
  3. 限制合法提案个数（挖矿难度），提高网络中共识的稳定性。
    权益证明
    在BTC中没有运用，ETH计划从PoW迁移到PoS。

权益证明（Proof of Stake，PoS）类似于现实生活中的股东机制，拥有股份越多的人越容易获取记账权。

PoS 试图解决在 PoW 中大量资源被浪费的问题，一般来看在PoS机制下掌握全网超过1/3资源才能左右结果（三人投票情景）。

UTXO

由于比特币是去中心化的，每次交易发生过后，需要网络确认即可认为交易完成。

  • 未经使用（引用）的交易的输出（Unspent Transaction Outputs，UTXO）可以被新的交易引用作为其合法的输入
  • 被使用过的交易的输出（Spent Transaction Outputs，STXO），则无法被引用作为合法输入。
交易 目的 输入 输出 签名 差额
T0 A 转给 B 他人向 A 交易的输出 B 账户可以使用该交易 A 签名确认 输入减输出，为交易服务费
T1 B 转给 C T0 的输出 C 账户可以使用该交易 B 签名确认 输入减输出，为交易服务费
… X 转给 Y 他人向 X 交易的输出 Y 账户可以使用该交易 X 签名确认 输入减输出，为交易服务费
放进网络中的交易（深度为 0）并非是实时得到确认的。进入网络中的交易存在被推翻的可能性，一般要再生成几个新的区块后（深度大于 0）才认为该交易被确认。

相比于账户模型，UTXO模型优势在于并行处理和隐私保护，交易路径完整，劣势在于存储压力较大。

智能合约

比特币只有全网每秒7笔左右的交易数据，等待6个块“上链”则需要1小时左右确认时间。因此，闪电网络主要通过引入智能合约的思想来完善链下的交易渠道，核心主要为两个：RSMC（Recoverable Sequence Maturity Contract）和 HTLC（Hashed Timelock Contract）。前者解决了链下交易的确认问题，后者解决了支付通道的问题。闪电网络通过对这两个智能合约的机制整合，实现交易链下完成。
